Hello AWS KMS!
Field level encryption

What do we want?

Field level encryption for super sensitive data, such as

  • Credit Cards,
  • Personally Identifiable Information (PII),
  • Naked photos of yourself

Components

  • The data,
  • Cipher, such as AES256, PGP, RSA, choose your own adventure,
  • Encryption library (don't write your own crypto, just don't),
  • Customer Master Key (CMK), used to encrypt the data key (4kb data max),
  • Data key - use a new data key per piece of encrypted data
  • The Data Key

Envelope Encryption

Pros & Cons

Pros and Cons of using AWS KMS for field Level encryption

Cons

  • Data portability - vendor lock-in-sorta,
  • Extra latency - more data to store & transfer

Pros

  • Secure - Attestations for SOC 1,2,3, PCI, ISO, HIPAA and more,
  • Easy (comparatively),
  • Regular enhancements,
  • Leverages IAM Roles,
  • Lower latency, not posting/returning large data packets,
  • Data Portability using multiple CMKs
